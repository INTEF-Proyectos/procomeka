# Epic: Cumplimiento del Esquema Nacional de Seguridad (ENS)

## Contexto
El pliego de licitación CE01698/2023 (PPT-SDA-26-2021) establece como requisito no funcional el cumplimiento del Esquema Nacional de Seguridad (ENS), regulado por el Real Decreto 311/2022.

## Objetivos
- Categorizar el sistema según las dimensiones de seguridad del ENS.
- Implementar medidas técnicas de seguridad (Análisis estático, Auditoría, Protección de datos).
- Generar la documentación requerida para el cumplimiento normativo.

## Requisitos funcionales y no funcionales
- **ENS-REQ-01**: Categorización del sistema (Básico/Medio/Alto).
- **ENS-REQ-02**: Análisis de riesgos y declaración de aplicabilidad.
- **ENS-REQ-03**: Configuración de GitHub Advanced Security (CodeQL, Dependabot, Secret Scanning).
- **ENS-REQ-04**: Registro de actividad completo (Trazabilidad).
- **ENS-REQ-05**: Registro de accesos a la API.
- **ENS-REQ-06**: Cumplimiento RGPD/LOPDGDD.
- **ENS-REQ-07**: Plan de continuidad del servicio.

## Usuarios afectados
- Administradores del sistema
- Curadores editoriales
- Autores
- Usuarios finales (lectores)
- Auditores de seguridad
